Font-face researcher

  1. There are several types of font formats:
    • WOFF
    • TTF
    • SVG
    • WOFF2
    • OTF
    • EOT
  2. The good candidat fontfile formats to use in all browsers is WOFF.
  3. The font-face syntax is also different to other @-blocks.The differences between a @font-face block and @keyframes / @media queries / @supports blocks is @font-face has its source property, fonts.
  4. Some fonts come with licencing restrictions, which means we can't use it for free but we have to pay to the invetor.
  5. We need eight font-face blocks to provide if we want to support all the chosen fonts.
  6. At roughly 20KB per font file, each of our users have to download 160kilobytes of fonts
  7. If we chose to offer WOFF2 as well as WOFF, would we use the (...)format(...), syntax. We would have 16 files, according to question 5.
  8. Most of our users would probably have the exact font we want to serve already installed on their devices. To make sure that the user's browser doesn't download the fonts they might already have, we use the local()/system.
  9. Sometimes we have to assign fallback fonts for our custom webfonts we add via `@font-face`. We should keep in mind that when it comes to selecting the fallback, everyone of the fallback font should have the same character.
  10. The browser can use one of several strategies for loading the fonts, e.g. showing nothing until the font is downloaded, or using a fallback font first. Font-display: swap is property that we can use to help the browser decide on which strategy to use.
